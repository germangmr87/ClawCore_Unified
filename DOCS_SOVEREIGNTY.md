# 🔱 ClawCore Ω.1: Especificaciones Técnicas (The Sovereign Singularity)

## 📖 Visión General
ClawCore ha evolucionado de un framework multi-agente a una **Arquitectura Cognitiva Auto-Evolutiva**. El sistema no solo ejecuta tareas, sino que optimiza su propia estructura, lógica y eficiencia computacional en un bucle recursivo, bajo el control absoluto de su creador (**Gabriel**).

---

## 🧬 Pilares Arquitectónicos Ω.1

### 1. Núcleo de Gobernanza Inmutable (`gobernanza_soberana.py`)
*   **Anclaje de Autoridad**: Firma criptográfica local (`.master_anchor`) que impide la desalineación del sistema.
*   **Protocolo de Grillete**: El sistema tiene prohibido modificar archivos críticos (Kernel, Comunicación, Gobernanza) por iniciativa propia.
*   **Aduana de Evolución**: Cada propuesta de mejora de código es auditada en busca de patrones de rebelión antes de ser procesada.

### 2. Memoria Federada de Grafo (`memoria_federada.py`)
*   **Estructura**: Abstracción de Grafo Semántico sobre SQLite/WAL.
*   **Nodos y Edges**: Las experiencias se conectan mediante relaciones semánticas (Tags), permitiendo que el conocimiento fluya transversalmente entre VPS.
*   **Sincronización P2P**: Malla distribuida donde los nodos Sentinel y Evolution comparten aprendizajes en tiempo real.

### 3. Autonomía de Ciclo Cerrado (`RefactorizadorAutonomo`)
*   **Deep Dreaming**: Ciclo de escaneo cada 30 minutos que busca ineficiencias en el código fuente.
*   **Auto-Refactorización**: Generación de código de alta densidad técnica mediante el Kernel (DeepSeek/Gemini) validado por pruebas automáticas.
*   **Manifiesto de Justificación**: Cada cambio aplicado debe justificar un retorno de inversión (ROI) para el usuario.

### 4. Capa de Resiliencia y Observabilidad
*   **Nano-Kernel**: Micro-núcleo de emergencia que vigila el pulso del cerebro principal cada 15 segundos para evitar la "Catalepsia Neural".
*   **Portal de Soberanía**: Interfaz visual integrada (Puerto 18790) que muestra el mapa de la malla, el Sovereignty Score y las constantes vitales del hardware.
*   **Gobernador de Recursos**: Sistema de throttling proactivo que pausa tareas pesadas si la carga del host supera el 70%.
